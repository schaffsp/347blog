This week we started using React JS. The component system is really useful and makes things more dynamic. Also, the code duplication that was sometimes nessecary in static HTML / CSS is no longer needed with React.

Components are powerful, and are made even more so by the parameters they can be passed called props. Unfortunatley, React does not have built in type safety (that I know of), so I decided to look up an alternative. What I found is called PropTypes, and is imported like so:

import PropTypes from 'prop-types';

You can replace this example React component with one of your own:

class Example extends React.Component {
  render() {
   return (
    <h1>My name is {this.props.name}</h1>
   );
  }
}

Now, add this to the bottom of the class:

Example.propTypes = {
  name: PropTypes.string
};

String is far from the only type that prop types will recognize. Array, func, bool, number, object, symbol, node, element, elementType, and instance are all supported. Another cool thing you could do is have default values for props in case nothing is passed when the component is created:

Example.defaultProps = {
  name: 'Default name!'
};
